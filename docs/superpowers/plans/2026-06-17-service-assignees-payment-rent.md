# Service Assignees/Vendors + Optional Payment Rent — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make rent optional/deletable on `/payments/new`, and replace the single free-text `담당자/업체` field on `/services` with a structured staff multi-select, an external-vendor picker (name + phone, autocompleting), and a "landlord did it themselves" flag.

**Architecture:** Feature 1 is a client-only change to one component. Feature 2 adds two tables (`service_request_assignee` junction, `service_vendor` lookup) plus two `service_request` columns, then threads them through the server actions, a new shared form subcomponent, and the list/detail pages. Vendors dedupe by name (upsert), so the client only sends name + phone.

**Tech Stack:** Next.js 16 (App Router, RSC + server actions), React 19, Kysely + Postgres, base-ui primitives (`@/components/ui/*`), Tailwind, shadcn-style components.

/**
 * AS 담당자/외부 업체 구조화.
 *  - service_request_assignee: 담당자(우리 직원) N:M (calendar_event_attendee 패턴)

 *  - service_request.landlord_self: 임대인 직접 처리 여부

 * 기존 자유입력 assignee 컬럼은 보존(legacy 표시) — 더 이상 폼에서 쓰지 않는다.
 */
export async function up(db: Kysely<unknown>): Promise<void> {
  await db.schema
    .createTable("service_vendor")
    .addColumn("id", "serial", (col) => col.primaryKey())
    .addColumn("name", "varchar", (col) => col.notNull().unique())
    .addColumn("phone", "varchar")
    .addColumn("created_at", "timestamptz", (col) =>

    .execute();

    .createTable("service_request_assignee")
    .addColumn("id", "serial", (col) => col.primaryKey())

    )
    .addColumn("user_id", "integer", (col) =>
      col.notNull().references("user.id"),
    )
    .addColumn("created_at", "timestamptz", (col) =>

      "user_id",

    .execute();

      col.references("service_vendor.id"),

```ts
function parseAssigneeIds(raw: FormDataEntryValue | null): number[] {
  if (typeof raw !== "string" || !raw) return [];
  try {
    const parsed = JSON.parse(raw);
    if (!Array.isArray(parsed)) return [];
    const ids = parsed
      .map((v) => Number(v))
      .filter((n) => Number.isInteger(n) && n > 0);
    return Array.from(new Set(ids));
  } catch {
    return [];
  }
}

// Upsert an external vendor by name; returns its id (or null when no name).
// Name is the source of truth so a free-typed entry resolves correctly; a
// changed phone updates the stored one.
async function resolveVendor(
  trx: Transaction<DB>,
  vendorName: string | null,
  vendorPhone: string | null,
): Promise<number | null> {
  if (!vendorName) return null;
  const existing = await trx
    .selectFrom("service_vendor")
    .select(["id", "phone"])
    .where("name", "=", vendorName)
    .executeTakeFirst();
  if (existing) {
    if (vendorPhone && vendorPhone !== existing.phone) {
      await trx
        .updateTable("service_vendor")
        .set({ phone: vendorPhone })
        .where("id", "=", existing.id)
        .execute();
    }
    return existing.id;
  }
  const created = await trx
    .insertInto("service_vendor")
    .values({ name: vendorName, phone: vendorPhone })
    .returning("id")
    .executeTakeFirstOrThrow();
  return created.id;
}
```
